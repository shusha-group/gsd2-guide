## Asides

Starlight provides four aside variants for callout content.

:::note
GSD auto-mode executes tasks sequentially within a slice. Each task receives the carry-forward context from its predecessors, ensuring continuity across the execution chain.
:::

:::tip
Use `npm run dev` during development — it starts the prebuild script automatically before launching the Astro dev server, so your generated content stays in sync.
:::

:::caution
The prebuild script overwrites files in `src/content/docs/` that match its generated manifest. Do not manually edit files that appear in `.generated-manifest.json` — your changes will be lost on the next build.
:::

:::danger
Running `rm -rf .astro/` clears the build cache and forces a full rebuild. Only do this when you encounter stale-cache errors — it adds significant rebuild time on large sites.
:::

## Steps

<Steps>

1. **Scout the codebase**

   The scout agent reads the relevant files and produces a context document for the planner.

2. **Plan the slice**

   The planner decomposes the work into ordered tasks with file lists, verification criteria, and observability signals.

3. **Execute each task**

   Worker agents implement each task sequentially, carrying forward context from completed predecessors.

4. **Verify the slice**

   All slice-level verification checks must pass before the slice is marked complete.

</Steps>
