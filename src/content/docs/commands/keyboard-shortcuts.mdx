## What It Does

GSD registers three keyboard shortcuts for actions that benefit from instant access. These work during any active session — whether you're in auto mode, reviewing architecture, or watching a build.

All shortcuts use the `Ctrl+Alt` modifier combination and require the [Kitty keyboard protocol](https://sw.kovidgoyal.net/kitty/keyboard-protocol/) for reliable detection. In terminals that don't support the protocol, GSD shows a slash-command fallback hint in the shortcut description rather than hiding the feature.

Pressing `Escape` during an active auto-mode session has a separate effect: it aborts the current agent turn and the `agent_end` handler automatically calls pause, so auto mode suspends with state preserved.

## Shortcuts

| Shortcut | Action | Fallback |
|----------|--------|----------|
| `Ctrl+Alt+G` | Open GSD dashboard | `/gsd status` |
| `Ctrl+Alt+V` | Toggle voice mode | `/voice` |
| `Ctrl+Alt+B` | Open background process manager | `/bg` |
| `Escape` | Abort current turn (pauses auto mode) | `/gsd pause` |

### Ctrl+Alt+G — GSD Dashboard

Opens a full-screen overlay showing milestone progress, active slice and task, completed units, timing, and the activity log. Same view as [`/gsd status`](../status/), but toggled in place without leaving the conversation. Press `Escape`, `Ctrl+C`, or `Ctrl+Alt+G` again to close.

Only activates when a `.gsd/` directory is present in the current working directory. If not found, a notice is shown instead.

### Ctrl+Alt+V — Voice Mode

Toggles real-time speech-to-text transcription. Requires a microphone. Same as running `/voice`.

### Ctrl+Alt+B — Background Process Manager

Opens an overlay listing all running background shell processes — dev servers, file watchers, build processes. Shows process ID, command, status, and uptime. Same as running `/bg`. Press `Escape` or `q` to close.

### Escape — Abort and Pause

When auto mode is running, pressing `Escape` aborts the current agent turn immediately. GSD's `agent_end` hook detects the `aborted` stop reason and calls pause automatically, suspending auto mode with all state preserved. Run `/gsd auto` to resume from where it left off.

This is distinct from [`/gsd pause`](../pause/), which sets a flag to pause cleanly after the current unit finishes rather than aborting mid-turn.

## Terminal Compatibility

These shortcuts rely on the **Kitty keyboard protocol**, a modern terminal extension that enables precise detection of modifier key combinations. Terminals that support it include:

- **Kitty** — Full support (protocol originator)
- **WezTerm** — Full support
- **Ghostty** — Full support
- **iTerm2** — Supported in recent versions

### Terminals Without Support

In terminals that don't implement the Kitty protocol, `Ctrl+Alt` key combinations can't be reliably detected. GSD detects this at startup and appends a fallback hint to the shortcut description — shortcuts that would be silently swallowed show their slash-command equivalent instead.

Known unsupported terminals:

- **macOS Terminal.app** (`TERM_PROGRAM=apple_terminal`) — No Kitty protocol support
- **Warp** (`TERM_PROGRAM=warpterm`) — Not supported
- **JetBrains IDE terminals** — Built-in terminals don't support the protocol

| Instead of | Use |
|-----------|-----|
| `Ctrl+Alt+G` | `/gsd status` |
| `Ctrl+Alt+V` | `/voice` |
| `Ctrl+Alt+B` | `/bg` |
| `Escape` | `/gsd pause` |

:::note
GSD auto-detects terminal capabilities at startup using `TERM_PROGRAM` and `TERMINAL_EMULATOR` environment variables. You don't need to configure anything — if your terminal supports the protocol, shortcuts just work. If it doesn't, fallback commands are always available.
:::
