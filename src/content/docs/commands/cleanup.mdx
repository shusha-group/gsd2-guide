## What It Does

`/gsd cleanup` moves phase directories from `.planning/phases/` into per-milestone archive folders under `.planning/milestones/`. It's a housekeeping command you run after one or more milestones have shipped but their phase directories were left in place.

When you run [`/gsd complete-milestone`](../complete-milestone/), you're offered the option to archive phase directories on the spot. If you skipped that step — or want to archive directories from multiple past milestones at once — `/gsd cleanup` handles it retroactively.

The command shows a dry-run summary before making any changes. Nothing moves until you confirm. If all completed milestones already have phase archives, or if there are no phase directories left to move, cleanup exits with a clear message and does nothing.

## Usage

```
/gsd cleanup
```

Cleanup takes no subcommands or arguments.

### Step-by-Step

1. **Read MILESTONES.md** — Parses `.planning/MILESTONES.md` to get a list of completed milestone versions (e.g., `v1.0`, `v1.1`).

2. **Check for existing archives** — Lists `.planning/milestones/` and filters out milestones that already have a `v{X.Y}-phases/` directory. Only milestones without an existing phases archive proceed.

3. **Determine phase membership** — For each unarchived milestone, reads its roadmap archive at `.planning/milestones/v{X.Y}-ROADMAP.md` to extract the phase numbers and names that belong to it.

4. **Match existing directories** — Checks which of those phase directories still exist in `.planning/phases/`. Directories that have already been removed or moved elsewhere are skipped.

5. **Show dry-run summary** — Presents a summary listing which phase directories will be moved to which destination for each milestone. No changes happen yet.

6. **Confirm** — Asks: *"Proceed with archiving?"*. Selecting Cancel stops without changes.

7. **Move directories** — Creates `.planning/milestones/v{X.Y}-phases/` for each milestone and moves the matching phase directories into it.

8. **Commit** — Commits the changes with message `chore: archive phase directories from completed milestones`, staging both `.planning/milestones/` and `.planning/phases/`.

### Early Exit Conditions

Cleanup stops without error when there's nothing to do:

- **All milestones already archived** — Reports "All completed milestones already have phase directories archived. Nothing to clean up."
- **No phase directories remain** — Reports "No phase directories found to archive. Phases may have been removed or archived previously."

## What Files It Touches

### Reads

| File | Purpose |
|------|---------|
| `.planning/MILESTONES.md` | Identifies completed milestones and their versions |
| `.planning/milestones/v{X.Y}-ROADMAP.md` | Determines which phases belong to each milestone |
| `.planning/phases/` | Lists phase directories available to archive |

### Creates

| File | Purpose |
|------|---------|
| `.planning/milestones/v{X.Y}-phases/` | Archive directory created per milestone |

### Writes

| File | Purpose |
|------|---------|
| `.planning/milestones/v{X.Y}-phases/{phase-dir}/` | Phase directories moved here from `.planning/phases/` |

## Examples

Archiving two milestones' worth of phase directories:

```
> /gsd cleanup

## Cleanup Summary

### v1.0 — MVP
These phase directories will be archived:
- 01-foundation/
- 02-auth/
- 03-core-features/

Destination: .planning/milestones/v1.0-phases/

### v1.1 — Security
These phase directories will be archived:
- 04-security/
- 05-hardening/

Destination: .planning/milestones/v1.1-phases/

Proceed with archiving? [Yes — archive listed phases / Cancel]

Archived:
- v1.0: 3 phase directories → .planning/milestones/v1.0-phases/
- v1.1: 2 phase directories → .planning/milestones/v1.1-phases/

.planning/phases/ cleaned up.
```

Nothing left to do:

```
> /gsd cleanup

All completed milestones already have phase directories archived. Nothing to clean up.
```

Phase directories were deleted manually before cleanup ran:

```
> /gsd cleanup

No phase directories found to archive. Phases may have been removed or archived previously.
```
