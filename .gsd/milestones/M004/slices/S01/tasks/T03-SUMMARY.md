# T03: Wire update.mjs, remove SDK, and run integration proof

**Replaced ANTHROPIC_API_KEY guard with findClaude(), removed @anthropic-ai/sdk dependency, and proved claude -p regenerates capture.mdx with correct structure (claude-sonnet-4-6, 139.7s).**

## What Happened

Three changes shipped:

1. **update.mjs wiring**: Added `findClaude` to the existing `regenerate-page.mjs` import. Replaced the `ANTHROPIC_API_KEY` env var guard in `runRegenerateStale()` with `findClaude()`. Updated the success log line from SDK-era `inputTokens/outputTokens/$cost` to subprocess-era `model/durationMs` since token counts aren't available from the subprocess interface. Updated the summary line to remove the cost total.

2. **SDK removal**: Deleted `@anthropic-ai/sdk` from `package.json` devDependencies, ran `npm install` to regenerate the lockfile (521 packages, down from 522+). Confirmed zero references to `@anthropic-ai/sdk` in scripts/, tests/, src/, package.json, and package-lock.json.

3. **Integration proof**: Ran `node scripts/lib/regenerate-page.mjs commands/capture.mdx` which spawned `claude -p` and regenerated the page. Claude (claude-sonnet-4-6) completed in 139.7s. The output has valid frontmatter and all 6 required sections (What It Does, Usage, How It Works, What Files It Touches, Examples, Related Commands). Astro build passed with 64 pages. All 20 tests pass.
