# T02: Run full `npm run update` pipeline with real page regeneration

**Slice:** S02 — Pipeline Integration and End-to-End Proof
**Milestone:** M004

## Description

This is the core proof for R049 (zero-intervention regeneration) and R052 (end-to-end on the 3 stale pages). The `npm run update` pipeline must detect stale pages, invoke `claude -p` for each, produce updated MDX that passes the Astro build, and have zero broken links.

The worktree currently has 37+ stale pages because `page-versions.json` was stamped before S01 changes. To avoid ~85 minutes of regeneration and focus on the 3 pages required by the roadmap, stamp all pages fresh first, then selectively mark only the 3 target pages as stale. The pipeline's final stamp step will re-stamp everything correctly.

Real `claude -p` invocations happen here. Budget ~140s per page × 3 pages ≈ 7 minutes for the regeneration step alone.

## Steps

1. **Stamp all pages fresh** to establish a clean baseline:
   ```bash
   node -e "import('./scripts/check-page-freshness.mjs').then(m => { const c = m.stampPages(); console.log('Stamped', c, 'pages'); })"
   ```

2. **Mark only the 3 target pages as stale** by editing `page-versions.json` — set the `contentHash` for `commands/config.mdx`, `reference/skills.mdx`, and `reference/extensions.mdx` to `"stale"` (a value that won't match any real hash). Verify:
   ```bash
   node -e "import('./scripts/check-page-freshness.mjs').then(m => { const { stalePages } = m.getStalePages(); console.log(stalePages.length, 'stale:', stalePages.map(p => p.page)); })"
   ```
   Should show exactly 3 stale pages.

3. **Run the full pipeline**:
   ```bash
   npm run update
   ```
   This runs all 9 steps. The regenerate step should:
   - Detect 3 stale pages
   - Find `claude` CLI via `findClaude()`
   - Invoke `claude -p` for each page (per-page, sequentially)
   - Log ✓ with model name and duration for each successful regeneration
   
   The build and check-links steps must pass after regeneration.

4. **Verify regeneration output** — Inspect the pipeline output log for:
   - `commands/config.mdx` → ✓
   - `reference/skills.mdx` → ✓
   - `reference/extensions.mdx` → ✓
   - Build step: 66+ pages
   - Check-links step: 0 broken links
   - Stamp step: stamped N pages

5. **Verify regenerated page quality** — Spot-check the 3 regenerated files:
   - Each has valid frontmatter (title + description)
   - `commands/config.mdx` has the 6 required sections (What It Does, Usage, How It Works, What Files It Touches, Examples, Related Commands) — though section names may vary slightly
   - Files are non-trivially different from empty/stub content
   
6. **Regression check** — Run the test suite:
   ```bash
   node --test tests/regenerate-page.test.mjs
   ```
   All 20 tests must still pass.
