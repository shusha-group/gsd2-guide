## Edge Cases

### Missing prompts.json (Section 6 guard)

```bash
# Temporarily rename prompts.json
mv content/generated/prompts.json content/generated/prompts.json.bak
node scripts/lib/build-page-map.mjs 2>&1 | grep "pages mapped"
mv content/generated/prompts.json.bak content/generated/prompts.json
```

**Expected:** Script runs without error and reports fewer pages (48), not 80. The `fs.existsSync` guard makes Section 6 a no-op when prompts.json is absent.

### Test suite catches missing command slug

The COMMAND_SLUGS array in `tests/page-map.test.mjs` must stay in sync with the actual `src/content/docs/commands/` directory. If a new command page is added without updating COMMAND_SLUGS, the "includes all 33 command pages" test will fail:

```bash
node --test tests/page-map.test.mjs 2>&1 | grep -A5 "includes all 33"
```

**Expected (with all 33 slugs in sync):** `ok 2 - includes all 33 command pages`

## Failure Signals

- `ls src/content/docs/prompts/*.mdx | wc -l` returns < 32 → one or more stub files are missing
- `grep -c "'/prompts/" astro.config.mjs` returns < 32 → sidebar entries are incomplete
- `node --test tests/page-map.test.mjs` shows `# fail > 0` → source-map structure is broken; check the failing test name for the specific assertion
- `npm run build` exits non-zero with an Astro error naming a `prompts/*.mdx` file → that file has invalid frontmatter or MDX syntax
- `python3 -c "... page-source-map.json prompts count"` returns < 32 → Section 6 in build-page-map.mjs failed silently; check if prompts.json exists and is valid JSON
- Build produces < 104 pages → stub page content broke Astro content collection validation
