# S03: Prompt page content generation — Research

**Date:** 2026-03-19

## Summary

S03 replaces the 32 stub MDX prompt pages (from S02) with fully authored content following the D056 structure: What It Does → Pipeline Position (Mermaid) → Variables (table) → Used By (links). All data needed for generation lives in `content/generated/prompts.json` (32 entries with name, slug, group, variables with descriptions, pipelinePosition prose, and usedByCommands arrays). The source prompt `.md` files provide the behavioral detail needed for the "What It Does" prose.

This is straightforward content generation — no new infrastructure, no new scripts, no new test patterns. The executor reads `prompts.json` + the source prompt file for each page, then writes the MDX directly. The main complexity is scale (32 pages) and maintaining consistent Mermaid diagram quality. Pages vary significantly in depth: `execute-task` has 16 variables and a 73-line source file; `guided-resume-task` has 2 variables and a 1-line source file. Guided variant pages should be shorter and explicitly note their delegation pattern.

## Recommendation

**Generate all 32 pages directly as the executor agent** — no generation script. Read `prompts.json` for structured metadata and each source prompt `.md` for behavioral understanding, then write each MDX file. Split into 3 tasks by group to keep each task's context window manageable:

1. **Auto-mode pipeline (10 pages)** — the most complex pages with full pipeline diagrams showing neighboring stages. These are the core loop prompts.
2. **Guided variants (8 pages) + Foundation (1 page)** — shorter pages. Guided variants note they delegate to the auto-mode version. `system` is a special case (no variables, no pipeline position in the dispatch sense).
3. **Commands (13 pages)** — command-invoked prompts with simpler diagrams showing the command trigger rather than the pipeline loop.

Verify with `npm run build` + `npm run check-links` after each task to catch issues early.

### Mermaid Styling Convention

Three fill colors from existing command pages:
- **Decision/highlight nodes:** `fill:#0d180d,stroke:#39ff14,color:#39ff14` — use for the "current prompt" node and key decision points
- **Action nodes:** `fill:#1a3a1a,stroke:#39ff14,color:#e8f4e8` — use for other pipeline steps
- **Error/terminal nodes:** `fill:#3a1a1a,stroke:#ff4444,color:#e8e8e8` — use for error exits (optional, most prompt diagrams won't need this)

### Link Format

- **Prompt → command:** `../../commands/{slug}/` (up from `/prompts/{slug}/` to root, then into commands)
- **Prompt → prompt (within same section):** `../{slug}/` (siblings)
- **Prompt → auto-mode page:** `../../commands/auto/` (for "see auto mode for the full pipeline")

## Constraints

- **Frontmatter `title` and `description`** must be valid YAML strings (no unescaped colons in title). Current stubs use bare prompt names as titles — keep this convention (e.g. `title: "execute-task"`, not `title: "/execute-task"`). Descriptions should be updated to meaningful one-liners.
- **Internal links** must use `../` prefix format per KNOWLEDGE.md. Prompt→command links need `../../commands/{slug}/`. Prompt→prompt links use `../{slug}/`.
- **Mermaid node IDs** must be valid (no hyphens in IDs — use camelCase like `ExecuteTask` not `execute-task`).
- **No raw prompt source dumps** per D029 — pages explain what prompts do, they don't show the prompt text.
- **32 prompts, 32 pages** — the stub count must remain exactly 32 after generation. No pages added or removed.

## Common Pitfalls

- **Mermaid node ID with hyphens** — Mermaid rejects IDs like `execute-task`. Use `ExecuteTask` or `ET` as the node ID, then set the display label: `ET["execute-task"]`.
- **Broken cross-links** — Prompt pages live at `/prompts/{slug}/`, command pages at `/commands/{slug}/`. The relative path from a prompt page to a command page is `../../commands/{slug}/`, not `../commands/{slug}/`. Getting this wrong produces broken links that only show up in `check-links`.
- **YAML title escaping** — Titles with colons or special characters need quoting. All current titles are simple hyphenated names, so this shouldn't be an issue, but watch for descriptions that contain colons.
- **Guided variant pages too long** — Some guided source files are 1-3 lines. The page content should be proportionally brief — don't pad with invented detail. A 15-20 line MDX body is fine for `guided-resume-task`.
