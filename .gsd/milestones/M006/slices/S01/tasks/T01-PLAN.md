# T01: Create solo-guide directory, 9 MDX files, and sidebar group

**Slice:** S01 — Guide Structure & Navigation
**Milestone:** M006

## Description

Create the `src/content/docs/solo-guide/` directory with 9 MDX files (1 index/landing page + 8 section stubs) and add a "Solo Builder's Guide" sidebar group to `astro.config.mjs`. This is pure structural scaffolding — the stubs contain placeholder text that subsequent slices (S02–S08) replace with full content.

The index page is substantive: it imports `CardGrid` and `LinkCard` from `@astrojs/starlight/components` and provides navigable links to all 8 sections. The 8 section stubs each have valid frontmatter and a short placeholder paragraph using Australian spelling.

The sidebar group goes at the end of the sidebar array in `astro.config.mjs`, just before the closing `],` (currently around line 169). Sidebar link values use `/solo-guide/{slug}/` format — Starlight's `base: '/gsd2-guide'` configuration handles the base path prefix automatically.

**Critical constraints:**
- Do NOT modify `content/generated/page-source-map.json` — solo-guide pages are excluded from the pipeline per decision D068
- Do NOT modify `src/content/docs/.generated-manifest.json` — it's auto-generated by prebuild
- Do NOT modify any files in `scripts/` — no pipeline changes needed
- Use Australian spelling throughout: behaviour, colour, recognise, organisation, organised, etc.
- All files must use `.mdx` extension (not `.md`) since they'll eventually use Starlight components
