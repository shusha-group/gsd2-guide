# S01 — Guide Structure & Navigation — Research

**Date:** 2026-03-19
**Depth:** Light — straightforward structural scaffolding using established codebase patterns.

## Summary

S01 creates the directory, sidebar group, index/landing page, and placeholder stubs for all 8 sections of the Solo Builder's Guide. The work is entirely pattern-following: `src/content/docs/recipes/` and `src/content/docs/user-guide/` already demonstrate the exact MDX frontmatter format, and the sidebar in `astro.config.mjs` already has 7 groups using the same `{ label, items: [{ label, link }] }` structure.

The three risks from the roadmap are all addressable with simple verification:

1. **Sidebar path mismatch** — sidebar `link:` values must use `/solo-guide/{slug}/` format (no base path prefix; Starlight's `base: '/gsd2-guide'` handles that).
2. **Pipeline contamination** — solo-guide files are safe because: (a) prebuild.mjs only copies from `content/generated/docs/` → `src/content/docs/`; (b) cleanup only removes files tracked in `.generated-manifest.json`; (c) manage-pages.mjs only operates on `commands/` and `prompts/` directories; (d) `content/generated/page-source-map.json` is not modified.
3. **Cross-reference format** — links from solo-guide pages to existing gsd2-guide pages use `../../page-name/` (up from `solo-guide/{slug}/` to docs root, then into the target page).

## Recommendation

Single task: create 9 MDX files in `src/content/docs/solo-guide/`, add one sidebar group to `astro.config.mjs`, run `npm run build`, confirm page count increases from 104 to 113 (9 new pages). No tests needed. No script changes. No pipeline changes.

### Stub Content Strategy

Each stub should have:
- Valid frontmatter (title + description)
- A brief placeholder paragraph (≥3 lines) indicating this section is coming — enough to pass the build and render a real page, but clearly not final content.
- The index page should be substantive: a brief intro to the guide's purpose and `LinkCard` links to all 8 sections.

Stubs must have enough content to be valid Starlight pages (frontmatter + at least one line of body). Empty body causes build warnings.

### Cross-Reference Link Format

From within `src/content/docs/solo-guide/daily-mix.mdx`, links to other gsd2-guide pages use:
- `../../commands/quick/` — up two levels from `solo-guide/daily-mix/` to docs root, then into `commands/quick/`
- `../../auto-mode/` — up two levels, then to root-level page
- `../why-gsd/` — sibling within solo-guide (up one level from `daily-mix/`)

This matches the KNOWLEDGE.md entry: "Starlight renders each .md page as /page/index.html. This means a page linking to a sibling needs `../sibling/`."
